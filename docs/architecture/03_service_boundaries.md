# 服务边界说明

## 控制面

### `gateway`

- 平台统一入口
- 负责鉴权、限流、路由、协议适配、请求观测注入

### `tenant-auth`

- 管理租户、应用、凭证、角色与策略
- 向其他服务提供鉴权与授权判断

### `model-registry`

- 管理模型元数据、部署形态、路由标签、版本、状态

### `task-orchestrator`

- 编排异步任务生命周期
- 负责分片、重试、补偿、死信恢复

### `index-manager`

- 管理索引创建、重建、压缩、迁移、回滚

### `cost-quota`

- 负责配额、预算、成本归因、账单汇总

## 数据面

### `ingestion`

- 对接外部数据源
- 将原始数据标准化为平台内部任务或文档对象

### `preprocess`

- 负责清洗、切块、去重、脱敏、多模态预处理

### `embedding-runtime`

- 负责同步推理、批量推理、动态批处理和模型适配

### `cache`

- 负责Embedding结果缓存和检索热点缓存

### `vector-store-proxy`

- 负责统一封装Milvus/Qdrant/OpenSearch等向量后端

### `retrieval`

- 负责ANN召回、过滤、融合、重排与结果组装

## 边界约束

- 控制面不直接执行重GPU推理
- 数据面不保存过多业务配置逻辑
- 每个服务只对外暴露稳定契约，不直接依赖其他服务内部表结构
- 跨服务调用优先通过API契约或事件协议，而不是共享数据库
