# Phase 0-1 交付计划

## 1. 文档目标

本文件将实施路线图进一步细化到 `Phase 0` 和 `Phase 1`，用于指导首轮项目启动、任务拆解、迭代排期和服务落地。

适用范围：

- 项目经理做排期和里程碑管理
- 架构师做边界和依赖评审
- 开发团队做首轮backlog拆分
- SRE和安全团队同步接入建设节奏

## 2. 交付策略

首轮建设遵循两个原则：

- 先形成“同步Embedding + 异步批任务 + 检索”最小闭环
- 从第一天开始预埋工业级能力的接口、日志、指标和租户边界

建议节奏：

- `Phase 0`：2~3周
- `Phase 1`：4~6周

总计建议首轮交付周期为 `6~9周`，具体可根据团队人数调整。

### 3.1 阶段目标

统一工程规范、技术选型和运行底座，让后续服务开发可以并行启动。

### 3.3 Phase 0 重点输出物

- 架构评审版文档
- OpenAPI初稿
- 事件命名规范和Topic规划
- 错误码规范
- 环境配置模板
- 服务脚手架模板
- CI/CD基础流水线

### 3.4 Phase 0 服务级任务

#### `gateway`

- 定义统一入口职责范围
- 冻结API版本策略和鉴权接入点
- 设计请求ID、租户ID、trace上下文透传规范

#### `embedding-runtime`

- 定义模型适配器接口
- 定义同步推理和批量推理抽象
- 明确第三方API模型与本地模型接入方式

#### `task-orchestrator`

- 定义任务状态机
- 定义任务分片、重试、幂等字段
- 定义异步回调和结果落点规范

#### `vector-store-proxy`

- 定义统一向量存储接口
- 冻结索引、collection、partition、metadata filter抽象
- 确定首期目标向量库

#### `preprocess`

- 定义文本切块、清洗、去重、脱敏接口
- 定义文档对象和chunk对象数据结构

### 3.5 Phase 0 验收标准

- 首批服务边界评审通过
- 首批API契约冻结
- 基础依赖环境可联通
- 所有服务具备统一初始化规范
- 可观测和安全字段已纳入接口或上下文设计

## 4. Phase 1：MVP闭环交付

### 4.1 阶段目标

交付一套可以联调、可试点、可演示的Embedding最小闭环，支持文本向量生成、向量写入和基础检索。

### 4.2 MVP范围边界

纳入范围：

- 文本Embedding
- 同步接口
- 异步批处理
- 文本预处理
- 向量写入
- topK检索
- 基础监控

暂不纳入范围：

- 多模态
- 控制台
- 完整计费系统
- 复杂A/B实验
- 高级重排

### 4.3 Phase 1 工作流

1. 客户端通过 `gateway` 发起请求
2. `gateway` 完成基础鉴权和限流
3. `cache` 查询请求或结果缓存
4. 未命中则调用 `embedding-runtime`
5. 批量数据通过 `task-orchestrator -> preprocess -> embedding-runtime`
6. 向量通过 `vector-store-proxy` 写入底层向量库
7. 查询通过 `retrieval` 返回topK结果
8. 指标、日志、追踪统一进入可观测系统

## 7. 建议迭代划分

### Iteration 0：基础就绪

- 完成API契约
- 完成基础依赖环境
- 完成服务模板
- 完成日志、指标、追踪规范

### Iteration 1：同步链路打通

- `gateway`
- `embedding-runtime`
- `cache`
- `tenant-auth`

目标：

- 打通同步文本Embedding
- 支持基础缓存和认证

### Iteration 2：检索闭环打通

- `vector-store-proxy`
- `retrieval`
- `preprocess`

目标：

- 打通文本入库和topK检索

### Iteration 3：异步任务打通

- `task-orchestrator`
- `preprocess`
- `embedding-runtime`

目标：

- 支持批量任务、状态查询和基础重试

### Iteration 4：MVP联调与试点准备

- 补齐集成测试
- 补齐监控与告警
- 准备演示数据和接入文档

目标：

- 形成可演示、可试点版本

## 8. 角色建议

### 控制面开发

### 安全与合规

- 负责鉴权方案、TLS、密钥、租户隔离规则评审

## 9. Phase 1 Definition of Done

满足以下条件视为Phase 1完成：

- 支持通过标准API获取文本Embedding
- 支持批量导入文本并完成向量入库
- 支持topK检索
- 支持最小租户识别和API认证
- 关键链路具备基础日志、指标、追踪
- 至少完成一轮端到端集成测试
- 至少有一个试点数据集可完成演示

## 10. 建议下一步

完成本计划后，建议立即进入以下动作：

1. 冻结 `Phase 0` 技术选型
2. 输出错误码、事件模型、任务状态机详细定义
3. 初始化 `gateway`、`embedding-runtime`、`task-orchestrator` 三个MVP服务模板

这三步完成后，项目就可以正式进入编码实施阶段。
