# SRE运行基线

## 每个服务上线前必须具备

- 健康检查：`liveness`、`readiness`
- 指标暴露：请求量、错误率、延迟、依赖状态
- 链路追踪：请求ID、租户ID、模型ID贯通
- 告警：P95/P99异常、错误率异常、资源异常
- 限流与熔断：避免级联故障
- Runbook：故障定位、回滚、重试、恢复步骤

## 重点故障场景

- 第三方模型API大量失败
- GPU资源耗尽或模型OOM
- Redis缓存雪崩
- 向量库分片异常
- Kafka堆积或死信增长

## 恢复策略

- 优先降级再扩容
- 优先隔离异常模型池
- 优先保证查询可用，其次保证离线任务吞吐
